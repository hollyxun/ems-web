# Casbin 策略管理 UI 使用手册

## 概述

策略管理页面提供 Casbin 权限策略的可视化管理功能，支持策略的增删改查、批量操作和权限测试。

**访问路径**：系统管理 → 策略管理 (`/manage/policy`)

---

## 页面结构

```
策略管理页面
├── 搜索栏（policy-search.vue）
│   ├── 角色选择器
│   ├── 资源路径筛选
│   └── 操作类型筛选
├── 操作栏
│   ├── 新增策略按钮
│   ├── 批量删除按钮
│   └── 刷新按钮
├── 策略列表表格
│   ├── 角色列
│   ├── 资源路径列
│   ├── 操作列
│   ├── 效果列
│   ├── 上下文列
│   └── 操作按钮列
└── 策略编辑抽屉（policy-operate-drawer.vue）
    ├── 基础信息
    ├── JSON 上下文编辑器
    └── 操作按钮
```

## 功能说明

### 1. 策略列表

表格展示所有 Casbin 策略：

| 列名 | 说明 | 示例 |
|------|------|------|
| 角色 | 策略关联的角色 ID | 1, 2, 3 |
| 资源路径 | API 路径或资源标识 | `/api/v1/user`, `*` |
| 操作 | HTTP 方法或操作类型 | `GET`, `POST`, `*` |
| 效果 | 策略效果 | `allow` / `deny` |
| 上下文 | 扩展权限信息（JSON） | `{"factoryIds":[1,2]}` |
| 操作 | 编辑、删除按钮 | - |

### 2. 搜索筛选

- **角色筛选**：下拉选择角色，显示该角色的所有策略
- **资源路径**：模糊匹配资源路径
- **操作类型**：筛选 GET、POST、DELETE 等操作

### 3. 新增策略

点击「新增策略」按钮，打开编辑抽屉：

**必填字段**：
| 字段 | 说明 | 格式 |
|------|------|------|
| 角色 | 选择角色 | 下拉选择 |
| 资源路径 | API 路径 | `/api/v1/module/action` |
| 操作 | HTTP 方法 | `GET`, `POST`, `PUT`, `DELETE`, `*` |
| 效果 | 允许/拒绝 | `allow` / `deny` |

**可选字段**：
| 字段 | 说明 | 格式 |
|------|------|------|
| 上下文 | 扩展权限配置 | JSON 对象 |

### 4. 编辑策略

点击表格行的「编辑」按钮，修改策略内容。

**注意**：修改策略会立即生效，影响该角色下所有用户的权限。

### 5. 删除策略

点击「删除」按钮，确认后删除策略。

**警告**：删除策略会立即撤销对应权限，可能导致用户无法访问相关功能。

### 6. 批量操作

选中多条策略后：
- **批量删除**：一次性删除多条策略
- **批量导出**：导出选中策略为 JSON

## JSON 上下文编辑

### 上下文字段说明

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `factoryIds` | `number[]` | 允许访问的工厂 ID 列表 | `[1, 2, 3]` |
| `factoryAll` | `boolean` | 允许访问所有工厂 | `true` |
| `shiftScope` | `string` | 班次数据范围 | `"own"` / `"all"` |
| `operations` | `string[]` | 操作权限列表 | `["view", "modify"]` |

### 示例配置

**允许访问所有工厂**：
```json
{
  "factoryAll": true,
  "shiftScope": "all"
}
```

**限制特定工厂 + 修改权限**：
```json
{
  "factoryIds": [1, 2, 3],
  "operations": ["view", "modify", "export"]
}
```

**仅查看本厂数据**：
```json
{
  "shiftScope": "own"
}
```

### 编辑器功能

- **语法高亮**：JSON 语法彩色显示
- **实时验证**：格式错误即时提示
- **快速模板**：预设常用配置模板

## 权限测试

使用「测试权限」功能验证策略效果：

1. 选择角色
2. 输入资源路径
3. 选择操作类型
4. 点击「测试」

系统返回：
- ✅ `allow` - 允许访问
- ❌ `deny` - 拒绝访问

---

## 注意事项

### Admin 角色

角色 ID = 1 的用户自动拥有所有权限，无需配置策略。

### 策略优先级

1. `deny` 策略优先于 `allow` 策略
2. 更具体的路径优先于通配符路径
3. 多个角色策略合并取并集

### 缓存刷新

策略变更后，系统会自动刷新权限缓存。如有延迟，可使用「清除缓存」功能手动刷新。

## 常见问题

### Q: 策略不生效怎么办？

1. 检查策略是否正确保存
2. 使用「测试权限」功能验证
3. 清除权限缓存
4. 用户重新登录

### Q: 如何配置按钮权限？

按钮权限策略格式：
```
资源路径: button:module:action
操作: access
```

示例：
```json
{
  "sub": "3",
  "obj": "button:user:create",
  "act": "access",
  "effect": "allow"
}
```

### Q: 如何批量配置相似策略？

1. 创建一条策略作为模板
2. 使用「复制」功能快速创建
3. 修改差异字段
